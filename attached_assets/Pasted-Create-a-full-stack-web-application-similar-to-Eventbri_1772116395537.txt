Create a full-stack web application similar to Eventbrite but tailored specifically for expats living in Russia. The app should allow users to create, discover, and purchase tickets for events in this niche. Use the following specifications:

**Tech Stack:**
- Frontend: Next.js (React) with Tailwind CSS for styling.
- Backend: Node.js with Express.js.
- Database: MongoDB (using Mongoose ODM).
- Authentication: NextAuth.js for email/password and Google OAuth.
- File storage: Cloudinary or local uploads (for event images).
- Payment: Simulate a mock payment flow (no real money handling).

**Core Features:**

1. **User Accounts:**
   - Sign up / login with email or Google.
   - Profile page showing user's created events and purchased tickets.
   - Role-based access: "Organizer" (can create events) and "Attendee" (can browse and buy tickets). All users start as attendees; they can become organizers when they create an event.

2. **Event Creation (Organizers):**
   - Form with fields: title, description, category (customized for the niche, e.g., "Wine Tasting," "Food Fair," "Cooking Class"), date & time, venue (address, city), image upload, and ticket types (name, price, quantity, max per order).
   - Preview before publishing.
   - Event management dashboard: edit, cancel, view attendees.

3. **Event Discovery (Attendees):**
   - Homepage with featured and upcoming events.
   - Search bar (by keyword) and filters: by date, location (city/region), category.
   - Event detail page with description, date/time, location map (embedded static map), ticket options, and an "Order Tickets" button.

4. **Ticket Purchasing:**
   - Select ticket quantity (respecting max per order and availability).
   - Simple cart (only one event at a time for MVP) with summary.
   - Checkout form (attendee name, email, optional notes) and a mock payment step (just a "Confirm Purchase" button).
   - After purchase, user receives a confirmation page with a downloadable PDF ticket (simple HTML-to-PDF generation, e.g., using jsPDF) and email notification (simulated with console log or a service like Nodemailer for demo).

5. **Responsive Design:**
   - Mobile-friendly, clean layout inspired by Eventbrite but with a color scheme and typography that appeals to **[your specific audience]** (e.g., warm colors for food events, modern tech style for corporate events).

**Additional Details:**
- Use environment variables for database connection, NextAuth secrets, etc.
- Provide a `README.md` with setup instructions (install dependencies, set up MongoDB, run dev server).
- The app should be ready to deploy on Replit – include a `.replit` configuration file if possible.

Write the code in a well-organized structure with comments. Focus on core functionality first; advanced features (like PDF tickets, real email) can be simplified for demonstration.
